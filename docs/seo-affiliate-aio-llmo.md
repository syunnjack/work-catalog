# SEO / AIO / LLMO / アフィリエイト導線設計

## 目的

多くの人に見つけてもらい、検索・AI検索・LLM回答・内部回遊から作品詳細へ誘導し、権利元または提携ASPの合法な販売/視聴ページへ送客する。

このサイトの勝ち筋は、動画や無許諾画像ではなく、**整理された作品メタデータ(品番・メーカー・レーベル・シリーズ・ジャンル・公式クレジット)と、作品間の内部リンクの網羅性**である。出演者の身元特定・素人名義出演の暴露は行わない。

## 基本方針

- SEO: Googleにクロール、理解、インデックスされやすいSSR/ISRページを大量に作る。
- AIO: Google AI Overviews / AI Modeで引用されやすい、短く明確な回答ブロックを各ページに置く。
- LLMO: LLMが作品、女優、メーカー、レーベル、シリーズ、タグの関係を抽出しやすいHTMLとJSON-LDにする。
- Affiliate: 作品詳細の検索意図を満たした後に、自然なCTAで公式/提携先へ誘導する。
- Compliance: アフィリエイトリンクは `rel="sponsored nofollow"`、UGCリンクは `rel="ugc nofollow"` を付与する。

## 公式ガイドラインに基づく設計根拠

- GoogleはAI検索でも基本SEOが有効としており、AI Overviews / AI ModeはSearch index上のクロール可能なページを使う。
- GoogleはAI検索向けに、独自性があり、読者に役立ち、整理されたコンテンツを推奨している。
- Googleは構造化データ、title、description、画像altなどの正確性を求めている。
- Googleは広告、スポンサー、paid placementのリンクに `rel="sponsored"` を使うよう案内している。

## ページ別AIO/LLMO最適化

### 作品詳細 `/works/[slug]`

最重要ページ。検索意図は「品番の作品情報を知りたい」「公式クレジットの出演者は誰か」「どこで見られるか」。

必須ブロック:

- Answer Block: `品番 XXX-000 の作品情報です。メーカーは X、レーベルは Y、シリーズは Z です。公式クレジットの出演者は A、B です。`(出演者が未クレジットの場合は「公式クレジットの出演者情報はありません」と明記する)
- Work Facts: タイトル、品番、メーカー、レーベル、シリーズ、発売日、収録時間、ジャンル。
- Official Availability CTA: 公式/提携先で確認、購入、視聴。
- Related Intent Links: 同女優作品(公式クレジットがある場合のみ)、同メーカー作品、同レーベル作品、同シリーズ作品、同ジャンル作品。
- FAQ: 「この作品のメーカー・レーベルはどこですか」「品番から検索できますか」「公式クレジットの出演者は誰ですか」「画像は許諾済みですか」。
- JSON-LD: `Movie`, `Person`(公式クレジットがある場合のみ), `Organization`, `BreadcrumbList`, `FAQPage`。

出演者が未クレジットの作品では、`OfficialInfoNotice`コンポーネントで「本サイトは独自に出演者を特定しません。最新情報は公式サイトでご確認ください」と明記し、当該プラットフォームの作品ページへ外部リンクする。

CTA配置:

- ファーストビュー下: 公式/提携先で見る。
- 関連作品下: 同レーベル作品を提携先で見る。
- FAQ下: 公式配信/販売ページへ。

### 女優詳細 `/actresses/[slug]`

検索意図は「女優名で出演作品を探す」「公式発表済みの旧芸名を確認する」。

このページは**公式クレジットのある作品を1件以上持つ女優のみ**生成する。未クレジット作品からの出演者補完によってページを増やすことはしない。

必須ブロック:

- Answer Block: `Aさんは公式クレジットで X 作品に登録されています。事務所公式発表の旧芸名は B です。`
- Alias Table: 現芸名、事務所公式発表済みの旧芸名・別名、ローマ字、読み方、活動期間(出典が公式のもののみ掲載)。
- Works Rail: 公式クレジットのある出演作品を、人気順・最近追加・メーカー別・レーベル別で表示。
- Affiliate CTA: この女優の公式クレジット作品を公式/提携先で探す。
- JSON-LD: `Person`, `BreadcrumbList`, `ItemList`, `FAQPage`。

### メーカー・レーベル・シリーズ詳細

検索意図は「このメーカー/レーベル/シリーズの作品一覧を見たい」。

必須ブロック:

- Answer Block: `Xレーベルは Yメーカー配下のレーベルです。主なシリーズは A、B です。`
- 階層リンク: メーカー > レーベル > シリーズ > 作品。
- 人気作品、最近追加。
- Affiliate CTA: このレーベルの作品を提携先で見る。

## アフィリエイト導線設計

### CTA種類

- Primary CTA: `公式/提携先で作品を確認`
- Compare CTA: `配信サイトで価格・在庫を比較`
- Actress CTA: `この女優の公式クレジット作品を探す`
- Label CTA: `このレーベルの人気作品を見る`
- Series CTA: `このシリーズをまとめて見る`

### 単品PPV主導線とサブスク副導線

このサービスの検索意図は「品番」「作品名」からの特定作品への興味が強い。そのため、作品詳細ページでは単品PPV、レンタル、単品購入を主導線にする。

一方で、同じメーカー/レーベル/ジャンルをまとめて見たいユーザーには、見放題サービスやサブスクリプションのアフィリエイト導線を副導線として表示する。

表示優先度:

1. 単品PPV、単品購入、レンタル
2. 価格比較、複数配信先
3. 見放題、サブスクリプション
4. 同メーカー、同レーベル、同シリーズの回遊導線(公式クレジットがある場合のみ同女優の回遊導線も追加)

DBでは `work_distribution_links.intent_type` で `ppv_primary`, `subscription_secondary`, `official_reference`, `price_compare` を分ける。

UX方針:

- 「単品PPVで確認」を作品ページの主CTAにする。
- 「見放題・サブスクでも探す」は下段に置く。
- サブスクCTAは、作品単体の掲載保証ではなく「関連作品を探す」文脈で表示する。
- サブスク対象作品、対象プラン、地域制限は送客先で確認するよう明記する。

### 中国・台湾・韓国ユーザー向け決済説明

中国圏ユーザーは銀聯（UnionPay）カード利用者が多く、台湾はLINE Pay、EasyWallet、悠遊卡などの生活決済、韓国はクレジットカード、Samsung Wallet/Samsung Pay、Naver Pay、Kakao Pay、Tmoneyなどのキャッシュレス決済利用が進んでいる。

ただし、現地で普段使う決済サービスがFANZA（DMM）で直接使えるとは限らない。作品詳細ページのアフィリエイトCTA直前に、国別の支払い案内を表示する。

案内方針:

- FANZA（DMM）は銀聯カードでの直接決済に対応していない場合がある。
- 利用可能な支払い方法は主にクレジットカードとDMMポイント払いである。
- 銀聯カード利用者には、銀聯に対応した決済サービスやウォレットを経由してDMMポイントをチャージし、DMMポイントで購入する流れを説明する。
- 台湾ユーザーには、台湾で一般的なLINE Pay、EasyWallet、悠遊卡などと、DMM側の支払い方法は別であることを説明する。
- 韓国ユーザーには、韓国はキャッシュレス利用が進んでいるが、韓国国内ウォレットがDMMで直接使えるとは限らないことを説明する。
- 台湾・韓国ユーザーには、国際ブランド付きクレジットカード、またはDMMポイント払いをまず確認するよう案内する。
- 決済方法は変更される可能性があるため、購入前にDMM公式ヘルプを確認するよう案内する。
- 簡体字中国語、繁体字中国語、韓国語の説明を併記する。

表示文言:

```txt
FANZA（DMM）は銀聯（UnionPay）カードでの直接決済に対応していない場合があります。銀聯カードを使う場合は、銀聯に対応した決済サービスやウォレットを経由してDMMポイントをチャージし、そのポイントで購入する流れを案内します。
```

簡体字:

```txt
FANZA（DMM）通常不能直接使用银联（UnionPay）卡付款。中国用户可以先通过支持银联的支付服务为DMM点数充值，然后使用DMM点数购买或观看作品。付款方式可能会变更，请以DMM官方帮助页面为准。
```

繁体字:

```txt
台灣使用者平常常用LINE Pay、EasyWallet、悠遊卡等支付方式，但FANZA（DMM）不一定能直接使用這些台灣在地支付工具。建議先確認是否可使用國際信用卡，或先儲值DMM點數，再以DMM點數付款。
```

韓国語:

```txt
한국은 신용카드와 Samsung Wallet/Samsung Pay, Naver Pay, Kakao Pay, Tmoney 같은 캐시리스 결제가 널리 사용됩니다. 다만 FANZA（DMM）에서는 한국 내 간편결제가 직접 지원되지 않을 수 있으므로, 사용 가능한 해외 결제 카드 또는 DMM 포인트 결제를 먼저 확인하세요.
```

### 表示ルール

- メタデータ本文を読める状態にしてからCTAを置く。
- CTAは作品確認の補助として置き、誤クリック誘導をしない。
- 広告表記を明示する。
- 外部リンクは直接リンクではなく `/go/[id]` 経由でクリックログを保存する。
- リンク属性は `rel="sponsored nofollow noopener"`。
- 未成年閲覧防止、年齢確認、成人向けコンテンツ注意を導線前に表示する。

## DB追加設計

- `affiliate_programs`: ASP、公式提携先、広告主。
- `distribution_platforms`: FANZA、SOD、MGS動画、DUGA/APEXなど国内大手配信/販売先。
- `payment_methods`: クレジットカード、DMMポイント、銀聯、LINE Pay Taiwan、EasyWallet、Samsung Wallet、Naver Pay、Kakao Pay、Tmoneyなど。
- `platform_payment_methods`: プラットフォーム別・地域別の決済対応メモ。
- `work_distribution_links`: 作品ごとの掲載URL、販売形態、地域メモ、価格メモ。
- `affiliate_links`: 作品/メーカー/レーベル/シリーズに紐づく送客URL(公式クレジットがある場合のみ女優にも紐づける)。
- `affiliate_click_logs`: クリック計測。
- `affiliate_conversions`: 成果計測。ASP側のpostbackまたはCSV取り込み。

## 国内大手プラットフォーム網羅

初期対応:

- FANZA / DMM
- SOD / SOD Prime / SOD Store
- MGS動画
- DUGA / APEX
- H-NEXT
- DLsite
- Sokmil
- FC2コンテンツマーケット

運用ルール:

- 公式または正規ASPで提携できるものだけを有効化する。
- 法務/掲載確認前は `legal_review_status = pending` とする。
- 海賊版、無許諾転載、出所不明なミラーサイトは登録しない。
- プラットフォームごとに決済方法、地域制限、ポイント利用可否を別管理する。
- 作品ページでは「この作品を公式/提携先で確認」として複数候補を比較表示する。

## LLMO向けHTML設計

- 主要情報はクライアントJSに閉じ込めず、SSR HTMLに出す。
- `h1` はページ主語、`h2` は質問/属性単位にする。
- 作品ページ冒頭に50〜120字の結論文を置く。
- テーブルまたは定義リストで、LLMが抽出しやすい属性名と値を並べる。
- 品番表記ゆれ、公式発表済みの旧芸名は本文と構造化データの両方に含める。
- ページ末尾にFAQを置く。
- 更新日、情報の出典(公式クレジット/メーカー提出)を表示する。

## AIO向けAnswer Blockテンプレート

```txt
{品番} の作品情報です。メーカーは {メーカー名}、レーベルは {レーベル名}、シリーズは {シリーズ名} です。公式クレジットの出演者は {女優名一覧} です（未クレジットの場合は「公式クレジットの出演者情報はありません」と表示）。
```

## 計測KPI

- Search Console: 表示回数、CTR、平均掲載順位、クエリ。
- GA/Vercel Analytics: 作品ページ滞在、内部リンククリック、検索実行。
- Affiliate: CTA表示回数、クリック率、成果率、EPC。
- LLMO: ChatGPT、Perplexity、Gemini、Google AI Modeからのreferral、指名検索増加。
- Database Quality: 品番検索ゼロ件率、メーカー提出情報の反映率、公式クレジット網羅率(登録作品のうちメーカー公式データと一致した割合)。

## 実装優先順位

1. 作品詳細のAnswer BlockとCTA。
2. `/go/[id]` リダイレクトとクリックログ。
3. `affiliate_programs`, `affiliate_links`, `affiliate_click_logs`。
4. `distribution_platforms`, `work_distribution_links`, `platform_payment_methods`。
5. FANZA、SOD、MGS動画、DUGA/APEXの作品別URL登録。
6. レーベル/メーカー/シリーズ別CTA。
7. FAQPageとBreadcrumbListのJSON-LD拡張。
8. `maker_submissions` 経由の公式情報提出フロー。
9. Search Consoleとアフィリエイト成果のダッシュボード。

## 多言語展開設計

対象言語:

- 日本語: `ja`
- 台湾向け繁体字中国語: `zh-tw`
- 中国向け簡体字中国語: `zh-cn`
- 韓国語: `ko`
- タイ語: `th`
- ベトナム語: `vi`
- インドネシア語: `id`
- 英語: `en`

初期実装:

- 画面上部に言語タブを表示する。
- URLの `?lang=` で選択言語を保持する。
- トップページ、検索UI、主要ナビゲーションから段階的に翻訳する。
- 作品名、女優名、メーカー名、レーベル名、シリーズ名は原典名を主表示し、翻訳説明文を補助表示する。

本格SEO展開時:

- `/zh-tw/works/[slug]`, `/ko/actresses/[slug]` のようなロケール別URLへ移行する。
- `hreflang` を各言語ページに出す。
- `canonical` は各言語ページ自身を指す。
- 言語別sitemapを分割する。
- DBに `localized_contents` を追加し、作品説明、FAQ、メーカー/レーベル説明を言語別に保存する。

LLMO多言語方針:

- 各言語でAnswer Blockを用意する。
- 品番、女優名、公式発表済みの旧芸名、メーカー、レーベル、シリーズは翻訳せず、説明文だけ翻訳する。
- 台湾、中国、韓国、東南アジアユーザー向けに「日本語名」「読み方」「ローマ字」「現地語説明」を並べる。
- AI翻訳は下書きに限定し、公開前にレビュー状態を保存する。
