# 収益導線を網羅したサイト設計

このドキュメントは**第一の核(新品配信・販売のアフィリエイト導線)**を扱う。**第二の核(廃盤・希少タイトルの中古相場情報)**は [`docs/used-market-pricing.md`](./used-market-pricing.md) を参照。作品詳細ページでは両方の導線を併記し、「新品で見る/買う」「中古相場を確認する」双方のユーザー意図に応える。

## 結論

このサイトは、検索流入を直接アフィリエイトへ送るのではなく、**作品・メーカー・レーベル・シリーズ**を経由して、ユーザーの検索意図(品番/作品名/メーカー名で探している)を解決した直後に最適な収益導線を出す。

前身の企画にあった「人物(タレント/チア/グラビアアイドル)を横断的に追跡・ランキングする」設計は採用しない。収益導線はあくまで**作品・カタログ起点**で設計する。

収益ページは以下の4階層で設計する。

```txt
Discovery: 品番検索 / 作品名検索 / メーカー検索 / レーベル検索
Intent: 作品詳細 / シリーズ / ジャンル
Decision: 比較 / 在庫 / 電子版 / 決済 / 海外対応
Conversion: /go/[id] / お気に入り / 通知 / 再訪問
```

## 収益導線マップ

| 検索意図 | 主ページ | 主CTA | 副CTA | 再訪問導線 |
| --- | --- | --- | --- | --- |
| 品番で探す | `/works/[slug]` | 単品PPV/購入 | 見放題で関連作品を探す | 同シリーズ通知 |
| 作品名で探す | `/works/[slug]` | 単品PPV/購入 | 価格比較 | お気に入り |
| メーカーで探す | `/makers/[slug]` | 人気作品を見る | 新着作品を見る | 新作通知 |
| レーベルで探す | `/labels/[slug]` | 人気作品を見る | シリーズ一覧 | 新作通知 |
| シリーズで探す | `/series/[slug]` | シリーズ作品を見る | 関連シリーズ | 新作通知 |
| 女優名で探す(公式クレジットのみ) | `/actresses/[slug]` | 公式クレジット作品を見る | 同メーカー作品 | 新作通知 |
| 海外から購入 | `/guides/payments/[region]` | 購入可能な電子版 | 決済方法を確認 | 言語別おすすめ |

## トップページ設計

トップページは、検索とランキングで発見させ、作品詳細へ送る。

表示順:

1. 検索ボックス: 作品名、品番、メーカー名、レーベル名、女優名。
2. 人気作品: 単品PPV導線が強い。
3. 新着作品: メーカー別、レーベル別。
4. メーカー/レーベル一覧: 主要メーカーへの導線。
5. 配信サイト一覧: 決済説明、PPV/サブスク比較。
6. 海外ユーザー向け購入ガイド: 中国、台湾、韓国。

## 作品詳細ページ設計

`/works/[slug]` が最重要ページ。

表示項目:

- タイトル
- 品番
- メーカー/レーベル/シリーズ
- 発売日
- 収録時間
- ジャンル
- 公式クレジットの出演者(未クレジットの場合は`OfficialInfoNotice`を表示)
- 許諾済みサムネイル
- 配信/販売先一覧(比較表)
- 関連作品(同メーカー/同レーベル/同シリーズ/同ジャンル)

CTA:

- 単品PPVで確認
- 見放題・サブスクでも探す
- 価格比較
- 同メーカー/同レーベルの作品を見る

JSON-LD:

- `Movie`
- `Person`(公式クレジットがある場合のみ)
- `Organization`
- `Offer`
- `BreadcrumbList`
- `FAQPage`

## メーカー・レーベル・シリーズページ設計

表示項目:

## ランキングページ設計

ランキングはCV前の強い中間ページにする。**個人(女優)単位の人気ランキングは設けない。**

```txt
/ranking/works/
/ranking/makers/
/ranking/labels/
/ranking/series/
```

ランキング軸:

- 閲覧数
- 検索数
- お気に入り数
- クリック率
- 成果率
- 新着作品数

CTA:

- 人気作品を公式で確認
- 新着メーカー/レーベルを見る

## 海外ユーザー向け設計

### 中国

- 簡体字ページを用意する。
- 銀聯カードが直接使えない可能性を説明する。
- DMMポイント、国際カード、電子版購入を優先案内する。

### 台湾

- 繁体字ページを用意する。
- LINE Pay、EasyWallet、悠遊卡は日本配信サイトで直接使えない可能性を説明する。

### 韓国

- 韓国語ページを用意する。
- Samsung Wallet、Naver Pay、Kakao Pay、Tmoneyが日本サイトで直接使えるとは限らないことを説明する。

## CTAコンポーネント設計

### `OfferCompareTable`

配信/販売先の比較表。

列:

- サービス名
- 販売形態
- 価格メモ
- 在庫/配信状況
- 海外対応
- CTA

### `PaymentRegionGuide`

中国、台湾、韓国ユーザー向けの決済説明。

表示:

- 対象地域
- 使える可能性がある方法
- 直接使えない可能性がある方法
- 公式ヘルプリンク
- 注意文

### `DisclosureNotice`

広告/アフィリエイト/権利表記。

表示:

- アフィリエイト広告を含むこと
- 外部リンクは公式/提携先であること
- 画像は許諾済みのみ掲載すること
- 在庫/価格/決済は送客先で確認すること

### `OfficialInfoNotice`

出演者情報が未クレジットの作品に表示する注記。

表示:

- 「本サイトは独自に出演者を特定しません」
- 公式サイトの作品ページへの外部リンク

## DB/API設計

### 追加テーブル候補

- `work_view_stats`: 作品単位の閲覧集計(ランキング算出用)。
- `notification_subscriptions`: 新作発売日通知。

### API

| Method | Path | 用途 | キャッシュ |
| --- | --- | --- | --- |
| GET | `/api/rankings?type=works\|makers\|labels\|series` | 作品/メーカー/レーベル/シリーズ単位のランキング | 5分 |
| POST | `/api/notifications` | 新作通知登録 | no-store |
| GET | `/go/[id]` | クリック計測リダイレクト | no-store |

## SEO / AIO / LLMO

### AIO向けAnswer Block

作品ページ:

```txt
{品番} は {メーカー名} の作品です。レーベルは {レーベル名}、シリーズは {シリーズ名} です。公式クレジットの出演者、配信/販売先、価格比較を確認できます。
```

メーカー/レーベルページ:

```txt
{名称} の人気作品、新着作品、関連レーベルを確認できます。
```

### 内部リンク

- 作品 → 同メーカー作品 → 同レーベル作品
- 作品 → 同シリーズ作品
- 作品 → 女優ページ(公式クレジットがある場合のみ) → 同女優の公式クレジット作品
- ランキング → 作品 → 配信/販売先比較

### 構造化データ

- 作品: `Movie` または `CreativeWork`
- 出演者: `Person`(公式クレジットがある場合のみ)
- 商品/購入: `Offer`
- メーカー/レーベル: `Organization`
- ランキング: `ItemList`
- FAQ: `FAQPage`

## 実装優先順位

1. `OfferCompareTable` と `/go/[id]` 計測を全CTA共通化。
2. 作品ページを作成(公式クレジット表示、未クレジット時の`OfficialInfoNotice`含む)。
3. メーカー/レーベル/シリーズページを作成。
4. 地域別決済ガイドを作成。
5. ランキングにクリック率、成果率を反映(作品/メーカー/レーベル単位)。
6. `maker_submissions` 経由の公式情報提出フローを実装。
7. 通知機能で再訪問を増やす。

## 運用ルール

- 公式または正規提携先だけを掲載する。
- 画像は許諾済みサムネイル、公式埋め込み、権利元提供素材のみ使う。
- 価格、在庫、決済、配送可否は変動するため「送客先で確認」と明記する。
- アフィリエイトリンクには `rel="sponsored nofollow noopener"` を付ける。
- 出演者情報は公式クレジットのみを掲載し、独自の身元特定・推測は一切行わない。
- 地域別決済案内は法務/ASP規約を確認してから公開する。
