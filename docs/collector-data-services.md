# B2Bデータ/API提供設計(第三の収益核)

## 目的

Core 1(新品配信アフィリエイト)・Core 2(中古相場)はいずれもクリック課金型のアフィリエイト収益であり、広告ネットワークの提携終了・単価変動に収益が左右されやすい。第三の核として、蓄積した構造化データ(作品カタログ・配信状況・中古相場)を**審査済みの法人向けにAPI/データフィードとして有償提供**し、月額/年額のサブスクリプション収益で収益源を分散する。

将来の拡張として、個人向けフリーミアム課金(コレクション管理・価格アラート)も構想しているが、今回のスコープでは扱わない(別途着手する)。

## 設計原則(境界)

- **提供先は審査済みの法人のみ**。個人向けの無制限API公開は行わない。
- 提供データは Core 1/Core 2 で扱う**作品・メーカー・レーベル・シリーズ・配信状況・中古相場の集計値**に限る。
- **ユーザーの個人データ(閲覧ログ、検索ログ、お気に入り、コメント投稿者情報)は一切提供しない。**
- **`maker_submissions`の未承認情報は提供しない。**
- 出演者に関するデータは、サイト本体に保存している**公式クレジットのみ**に限られる(そもそも非公式情報はDBに存在しない)。
- 利用規約に「本データを出演者個人の特定・アウティング目的で利用することを禁止する」条項を明記する。これはサイト全体の設計方針(出演者の身元特定機能を持たない)と直結する、契約上の担保。
- 用途審査を経ずにAPIキーを発行しない。規約違反が判明した場合は即時キー失効。

## 想定顧客

- 古物商許可を持つ中古販売店(自社の相場データ整備に利用)
- 他のアフィリエイトサイト/メディア運営者
- 市場調査会社

## 提供データの範囲

- 作品カタログ: 品番、タイトル、メーカー、レーベル、シリーズ、ジャンル、発売日
- 公式クレジット出演者名(提供する場合も契約条項で身元特定目的の利用を明示的に禁止する)
- 配信/販売プラットフォーム掲載状況(`distribution_platforms`, `work_distribution_links`の集計)
- 中古相場データ(`work_market_prices`, `work_rarity_notes`の集計値)

## 提供プラン

| プラン | 内容 | 想定顧客 |
| --- | --- | --- |
| Starter | 月次スナップショットCSVダウンロード | 小規模の中古販売店 |
| Pro | リアルタイムAPI、レート制限あり | メディア運営者、中規模販売店 |
| Enterprise | 専用エンドポイント、SLA、カスタムデータフィード | 市場調査会社、大手販売店 |

## API設計

- 認証: APIキー方式。契約単位で発行し、法人審査後にのみ発行する。
- レート制限: プランごとに上限を設定し、`data_partner_api_usage_logs`で計測する。
- ベースパス: `/api/v1/partner/...`(サイト本体のユーザー向けAPIとは名前空間を分離する)。

| Method | Path | 用途 | 認証 |
| --- | --- | --- | --- |
| GET | `/api/v1/partner/works` | 作品カタログ一覧(ページング) | APIキー必須 |
| GET | `/api/v1/partner/works/{productCode}` | 作品詳細 | APIキー必須 |
| GET | `/api/v1/partner/used-market/prices?productCode=` | 中古相場集計値 | APIキー必須(Pro以上) |
| GET | `/api/v1/partner/makers`, `/labels`, `/series`, `/genres` | マスタデータ一覧 | APIキー必須 |

## データベース設計

### `data_partners`

法人契約先マスタ。

- `id`
- `company_name`
- `contact_name`
- `contact_email`
- `plan`: `starter` / `pro` / `enterprise`
- `review_status`: `pending` / `approved` / `rejected` / `suspended`
- `contract_started_on`
- `contract_note`
- `created_at`, `updated_at`

### `data_partner_api_keys`

- `id`
- `partner_id`(`data_partners`参照)
- `key_hash`(平文キーは保存しない)
- `scopes`: text[]（`catalog_read`, `used_market_read` など）
- `status`: `active` / `revoked`
- `created_at`
- `revoked_at`

### `data_partner_api_usage_logs`

- `id`
- `partner_id`
- `api_key_id`
- `endpoint`
- `response_status`
- `called_at`

## 契約・コンプライアンス

- 利用規約に以下を必須条項として含める。
  - 提供データを出演者個人の特定・追跡・アウティング目的で利用しない。
  - 再配布・第三者への転売を禁止する(契約プランの範囲外利用の禁止)。
  - データ出典のクレジット表示義務(該当プランの場合)。
- 契約前に担当者の本人確認・利用目的のヒアリングを行う(審査制、自動発行しない)。
- 定期的に`data_partner_api_usage_logs`を監査し、異常なアクセスパターン(想定用途と乖離した大量取得等)を検知した場合は利用停止を検討する。

## 収益モデル

- 月額/年額のサブスクリプション課金(決済代行サービスを利用する想定)。
- Enterpriseプランは従量課金オプション(API呼び出し数ベース)を併用可能にする。

## 実装優先順位

1. `data_partners`, `data_partner_api_keys`テーブルと、admin画面での契約先登録・キー発行機能。
2. 読み取り専用の`/api/v1/partner/works`(カタログ情報のみ、価格データなし)からMVPを開始する。
3. `data_partner_api_usage_logs`によるレート制限・利用ログ記録。
4. 中古相場データのAPI(`/api/v1/partner/used-market/prices`)を追加する。
5. 決済代行との連携、契約管理の自動化。

## 運用ルール

- 個人(非法人)への提供は行わない。
- 用途審査を経ずにAPIキーを発行しない。
- 利用規約違反(身元特定目的の利用等)が判明した場合は即時キー失効・契約解除する。
- 提供データの範囲は本ドキュメントの「提供データの範囲」を超えない(個人データ・未承認情報は対象外)。
